## 핵심 답변
가장 큰 근본적 처리 차이는 **반환 값의 용도**입니다. `@Controller`는 주로 반환된 문자열 데이터를 바탕으로 `ViewResolver`에 전송해, 완성된 HTML 템플릿(뷰 화면, JSP/Thymeleaf)을 렌더링하고 조합해 반환하기 위해 목적으로 쓰입니다. 
반면 `@RestController`는 내부에 `@ResponseBody`가 포함된 애노테이션으로 뷰 템플릿이 아닌 JSON, 텍스트 원형(Data) 포맷 그 자체를 HTTP 바디에 때려 직송하기 위한 API 전담 성격을 지닙니다.

## 꼬리 질문
- **Q. 일반 @Controller에서 JSON 데이터를 보내려면 어떻게 해야 하나요?**
  - A: 원하는 전송 타깃 메서드(함수) 바로 위에 단순히 `@ResponseBody`를 붙이기만 하면 뷰 리졸버가 반응하지 않고 HttpMessageConverter가 발동해 객체를 JSON으로 직렬화해 화면 없이 넘겨줍니다.
